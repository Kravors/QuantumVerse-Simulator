This is the highest-impact task in Phase 2, enabling real-time physics for the RL discovery agent, anomaly detection, and interactive exploration.

**Approach**:
- **Training data**: 10⁶ geodesics generated by DifferentiableGeodesicIntegrator (Task 2.3)
- **Model**: MLP (256-256-256) with ReLU activations
- **Framework**: PyTorch (Python) for training, ONNX export for C++ deployment
- **Target**: <1ms inference, <1% endpoint error vs high-precision integrator
- **Validation**: Held-out metrics (Schwarzschild, Kerr, exotic) + symplectic regularization

#### Python Side (Training Pipeline)

**`python/datagen.py`**:
- `schwarzschild_metric()`: metric components (g_tt, g_rr, g_θθ, g_φφ)
- `christoffel_schwarzschild()`: analytical Christoffel symbols (non-zero components)
- `geodesic_equation()`: RHS of geodesic ODE (du^λ/dτ = -Γ^λ_μν u^μ u^ν)
- `rk4_step()`: 4th-order Runge-Kutta integrator (matches C++ version)
- `integrate_geodesic()`: full integration to τ_max
- `generate_schwarzschild_sample()`: single geodesic with random M, r, τ
- `generate_dataset()`: parallel generation of N samples → HDF5 file

**Dataset Format** (HDF5):
```
/initial_event      (N, 4)  float32  [t, r, θ, φ]
/initial_velocity   (N, 4)  float32  [u^t, u^r, u^θ, u^φ]
/metric_params      (N, P)  float32  [M, a, Q, ...] (P = num parameters)
/tau                (N,)    float32  proper time
/final_event        (N, 4)  float32  [t_f, r_f, θ_f, φ_f]
attrs['metadata']   JSON    generation config
```

**`python/train.py`**:
- `GeodesicDataset`: PyTorch Dataset wrapper (lazy HDF5 loading, train/val/test split)
- `GeodesicMLP`: neural network (Linear-ReLU-BN x3 → output)
  - Kaiming initialization for ReLU
  - BatchNorm for training stability
  - Gradient clipping (max_norm=1.0)
- `train_epoch()`: training loop with Adam, MSE loss, optional symplectic regularization
- `validate()`: validation loss
- `export_to_onnx()`: exports best model to ONNX with dynamic batch axis
- `train()`: full training loop with:
  - Learning rate scheduling (ReduceLROnPlateau)
  - Checkpointing (every epoch + best model)
  - Metadata saving (JSON with hyperparams, dataset, performance)

**Training Configuration**:
- Optimizer: Adam (lr=1e-3, weight_decay=1e-5)
- Batch size: 128
- Epochs: 100 (early stopping via LR scheduler)
- Loss: MSE(final_event_pred, final_event_true)
- Symplectic loss: disabled (λ=0) for now (future work)
- Validation split: 10% of data

## Mathematical Foundation

### Input/Output Specification

**Input vector** (dim = 9 + N_params):
```
x = [t₀, r₀, θ₀, φ₀,      # initial event (4)
     u^t₀, u^r₀, u^θ₀, u^φ₀,  # initial 4-velocity (4)
     τ,                    # proper time to integrate (1)
     θ₁, θ₂, ..., θ_N]    # metric parameters (N) e.g., M, a, Q
```

**Output vector** (dim = 4):
```
y = [t_f, r_f, θ_f, φ_f]  # final event after proper time τ
```

**Note**: The network predicts only the final position, not the full trajectory. This is sufficient for many applications (e.g., "where will this particle be after τ?"). For full trajectory prediction, a sequence model (RNN/ODE-Net) would be needed (future extension).

### Network Architecture

```
Input (9+N) → Linear(9+N → 256) → ReLU → BatchNorm → 
            Linear(256 → 256) → ReLU → BatchNorm →
            Linear(256 → 256) → ReLU → BatchNorm →
            Linear(256 → 4) → Output (final event)
```

**Total parameters**: ~200K (for 256-256-256) - very small, fast inference.

### Loss Function

Primary loss:
```
L_MSE = ||y_pred - y_true||²₂  (mean squared error over batch)
```

Optional regularization (future):
```
L_symplectic = λ * ||J_pred - J_true||²  (Jacobian symplecticness)
```
Currently λ=0 (disabled).

Total loss: `L = L_MSE + λ * L_symplectic`

### Training Data Generation

**Source**: DifferentiableGeodesicIntegrator (C++ Task 2.3) or Python reimplementation.

**Sampling strategy**:
- Mass M: uniform [5, 50] M☉ (solar masses) for Schwarzschild
- Radius r: log-uniform [6M, 100M] (outside horizon, wide range)
- Proper time τ: uniform [0.1, 10.0] (geometric units)
- Initial velocity: from rest at radius r (u^r=0, u^t = 1/√(1-2M/r))
- Orbits: equatorial (θ=π/2) for simplicity; can extend later

**Dataset size**: 10⁶ samples (80% train = 800K, 10% val = 100K, 10% test = 100K)

**Expected training time**: ~1-2 hours on GPU (RTX 3090), ~6-12 hours on CPU.

## Validation & Success Criteria

### Accuracy Targets

| Metric | Target | Measurement |
|--------|--------|-------------|
| Endpoint MSE | < 1e-4 (normalized) | vs high-precision RK4 (10K steps) |
| Endpoint error (mean) | < 0.1% of r | on test set |
| Inference time (CPU) | < 1 ms | single sample, i7-12700K |
| Inference time (GPU) | < 0.1 ms | RTX 3090 |
| Speed-up vs RK4 | > 100× | (5-10ms → <0.1ms) |

### Generalization Tests

Must perform well on:
- **Schwarzschild** (various M, r) - in-training distribution
- **Kerr** (M, a) - out-of-distribution but similar metric family
- **Exotic** (regular black holes, f(R)) - extreme generalization

If performance degrades on OOD metrics, will need data augmentation (multi-metric training).

### Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Dataset generation too slow (10⁶ samples × 1000 steps) | High | Parallelize with multiprocessing (Python multiprocessing.Pool); use C++ DifferentiableGeodesicIntegrator via pybind11 for speed |
| Model fails to generalize to Kerr/exotic metrics | Medium | Include multi-metric data in training set (Schwarzschild + Kerr + RN) |
| ONNX Runtime integration complex | Low | Use prebuilt binaries, follow official C++ API examples |
| Inference >1ms on CPU | Medium | Optimize network (prune, quantize to INT8), use OpenVINO or TensorRT |
| Training diverges or overfits | Medium | Add dropout, weight decay, early stopping; increase dataset size |
